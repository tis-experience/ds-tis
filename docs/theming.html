ThemingTheming

O design system usa uma paleta primária única (Default Blue) e alterna entre light e dark via tokens semânticos. Para trocar a paleta, sobrescreva os tokens primary/toned via um novo seletor data-theme — ou experimente interativamente no Theme Playground.The design system uses a single primary palette (Default Blue) and switches between light and dark through semantic tokens. To change the palette, override the primary/toned tokens via a new data-theme selector — or try it interactively in the Theme Playground.

Como funcionaHow It Works

O design system usa um atributo HTML no elemento raiz para controlar a aparência:The design system uses one HTML attribute on the root element to control appearance:

  • data-mode alterna entre tokens de superfície/texto light e dark.data-mode switches between light and dark surface/text tokens.

Todos os componentes respondem automaticamente porque referenciam tokens semânticos, não valores hardcoded.All components automatically respond because they reference semantic tokens, not hardcoded values.

Criando um tema personalizadoCreating a Custom Theme

Crie seu próprio tema adicionando um novo seletor data-theme que sobrescreve os tokens semânticos primário e secundário. Siga este padrão:Create your own theme by adding a new data-theme selector that overrides the primary and secondary semantic tokens. Follow this pattern: 

/* Custom "Sunset" theme — paleta laranja */
[data-theme="sunset"] {
  /* Primary — fill sólido (botões brand, focus ring, links) */
  --ds-brand-background-default:  #EA580C; /* Orange 600 */
  --ds-brand-background-hover:    #C2410C; /* Orange 700 */
  --ds-brand-background-active:   #9A3412; /* Orange 800 */
  --ds-brand-content-default: #FFFFFF;

  /* Toned — fill suave (botões toned, badges subtle) */
  --ds-toned-background-default:    #FED7AA; /* Orange 200 */
  --ds-toned-background-hover:      #FDBA74; /* Orange 300 */
  --ds-toned-content-default: #C2410C; /* Orange 700 */

  /* Links e focus ring usam primary */
  --ds-link-content-default: #EA580C;
  --ds-link-content-hover:   #C2410C;
  --ds-border-focus:         #EA580C;
  --ds-focus-ring-color:     #EA580C;
}

/* Dark-mode override do tema (cores invertem hierarquia) */
[data-theme="sunset"][data-mode="dark"] {
  --ds-brand-background-default:  #FB923C; /* Orange 400 */
  --ds-brand-background-hover:    #FDBA74; /* Orange 300 */
  --ds-brand-background-active:   #FED7AA; /* Orange 200 */
  --ds-brand-content-default: #1C1917;

  --ds-toned-background-default:    #7C2D12; /* Orange 900 */
  --ds-toned-background-hover:      #9A3412; /* Orange 800 */
  --ds-toned-content-default: #FB923C;

  --ds-link-content-default: #FB923C;
  --ds-link-content-hover:   #FDBA74;
  --ds-border-focus:         #FB923C;
  --ds-focus-ring-color:     #FB923C;
}

Depois aplique com <html data-theme="sunset">. Todo componente que usa --ds-brand-background-default adotará sua nova paleta automaticamente.Then apply it with <html data-theme="sunset">. Every component that uses --ds-brand-background-default will pick up your new palette automatically.

Theme PlaygroundTheme Playground

Para gerar um tema completo sem calcular contraste manualmente, use o Theme Playground. Escolha a cor brand, corner radius e fontes; o engine gera a paleta 50–950 com validação WCAG, preview de componentes reais e exportação em três formatos:To generate a full theme without hand-calculating contrast, use the Theme Playground. Pick brand color, corner radius, and fonts; the engine builds the 50–950 scale with WCAG checks, live component preview, and three export formats:

  • CSS — bloco [data-theme="…"] pronto para colar no seu CSS.CSS — [data-theme="…"] block ready to paste into your stylesheet.
  • JSON — config do theme engine (js/theme/) para scripts ou integrações.JSON — theme engine config (js/theme/) for scripts or integrations.
  • URL — link compartilhável com o estado codificado (?c=).URL — shareable link with encoded state (?c=).

O playground altera CSS variables em runtime — não escreve em tokens/*.json nem no Figma. Para persistir no repo canônico, aplique o CSS exportado ou sincronize tokens manualmente após revisão.The playground updates CSS variables at runtime — it does not write to tokens/*.json or Figma. To persist in the canonical repo, apply the exported CSS or sync tokens manually after review.
